Practica #5

Administracion de Memoria

Paginacion Segmentacion Swap Memoria Virtual

Concepto general

El sistema operativo gestiona la memoria principal asignandola y compartiendola entre procesos activos para mantener estabilidad y rendimiento. Esta gestion permite que multiples programas se ejecuten al mismo tiempo sin invadir el espacio de memoria de otros procesos.

5.1 Paginacion y Segmentacion

La paginacion divide la memoria en bloques de tamaño fijo llamados paginas, mientras que la segmentacion la divide en regiones de tamaño variable relacionadas con la logica del programa (codigo, datos, pila, etc.). Ambos enfoques permiten administrar mejor la asignacion y proteccion de memoria.

5.2 Intercambio (Swap)

El swap mueve temporalmente procesos o paginas desde RAM al disco para liberar espacio cuando la memoria fisica es limitada. Aunque permite continuar la ejecucion, un uso excesivo de swap degrada el rendimiento por la diferencia de velocidad entre RAM y almacenamiento secundario.

5.3 Memoria Virtual

La memoria virtual permite ejecutar programas que aparentan disponer de mas memoria de la que existe fisicamente. Con apoyo de la MMU y tablas de paginas, el sistema traduce direcciones virtuales a marcos de pagina en memoria real, aislando procesos y mejorando seguridad.

5.4 Codigo: Obtener tamano de pagina

Codigo Fuente (C)

Consulta del tamano de pagina con sysconf y getpagesize.

#include <stdlib.h>
#include <stdio.h>
#include <unistd.h>

int main()
{
    printf("Tamano de pagina:%d \n\n", (int)sysconf(_SC_PAGESIZE));
    printf("Tamano de pagina:%d \n\n", (int)getpagesize());
    return EXIT_SUCCESS;
}

Salida en terminal

Tamano de pagina:4096

Tamano de pagina:4096

Ambos metodos devolvieron 4096 bytes, lo que confirma el tamano de pagina del sistema en este entorno.

Lo que aprendi y como se podria mejorar

Aprendi que el tamano de pagina condiciona como se organiza la memoria virtual y que puede verificarse con distintas llamadas del sistema para corroborar resultados.

Se puede mejorar validando errores de cada llamada y comparando la salida con utilidades del sistema para confirmar consistencia en distintos entornos.

5.5 Codigo: Obtener informacion estadistica del sistema

Codigo Fuente (C)

Uso de la funcion sysinfo para recabar datos de RAM y Swap.

#include <stdio.h>
#include <sys/sysinfo.h>

int main() {
    struct sysinfo si;
    sysinfo(&si);
    printf("Total RAM: %ld KB\n", si.totalram / 1024);
    printf("Libre RAM: %ld KB\n", si.freeram / 1024);
    printf("Swap Total: %ld KB\n", si.totalswap / 1024);
    return 0;
}

Salida en terminal

Total RAM: 15991580 KB
Libre RAM: 8719976 KB
Swap Total: 4194300 KB

Este programa consulta al kernel para obtener metricas globales de memoria. Los datos sirven para evaluar el estado del sistema, detectar presion de memoria y decidir ajustes en la ejecucion de procesos.

Lo que aprendi y como se podria mejorar

Aprendi a interpretar RAM total, RAM libre y swap para describir el estado global del sistema y detectar cuando existe presion de memoria.

Se puede mejorar calculando porcentajes de uso y registrando muestras periodicas para analizar tendencias de consumo durante mas tiempo.

5.6 Lo que aprendi

Aprendi a diferenciar memoria fisica y memoria virtual, asi como a interpretar por que un sistema puede seguir funcionando aun con RAM limitada gracias al swap. Tambien comprendi que paginacion y segmentacion no solo son conceptos teoricos: impactan directamente en rendimiento, proteccion y organizacion de procesos.

5.7 Como mejorarlo

Para mejorar esta practica se puede ampliar el programa para mostrar memoria usada, porcentaje de ocupacion y evolucion en el tiempo mediante muestreo periodico. Tambien es recomendable comparar resultados con herramientas de sistema como free, vmstat o top para validar mediciones.
