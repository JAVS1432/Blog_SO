Contexto del proyecto

Enfoque y metodologia del portafolio

Este portafolio surge como una bitacora de practicas del curso de Sistemas Operativos en sexto semestre. Cada actividad fue organizada para responder una pregunta tecnica concreta y dejar evidencia del razonamiento que nos llevo a cada solucion.

La estructura se divide en tres rutas: manejo de Linux, ejecucion concurrente y comunicacion entre procesos. En lugar de presentar solo definiciones, priorizamos ejercicios que ponen a prueba el comportamiento real del sistema operativo en escenarios controlados.

En cada entrega registramos: idea base, codigo implementado, salida observada y reflexion tecnica. Esta forma de documentar nos permite justificar decisiones, detectar mejoras y mostrar dominio progresivo de los conceptos.

Rutas de trabajo

Tema 1

Linux en practica

Administracion de archivos, control de permisos y automatizacion con scripts orientados a resolver tareas cotidianas del entorno Unix.

3 entregas

Tema 2

Procesos e Hilos

Analisis de creacion de procesos con fork(), uso de hilos POSIX y control de regiones criticas por medio de mecanismos de sincronizacion.

3 entregas

Tema 3

IPC en accion

Integracion de pipes, memoria compartida y semaforos para construir intercambio de datos entre procesos con control de orden y consistencia.

3 entregas

Tema 4

Interbloqueo

Estudio de condiciones de bloqueo mutuo, escenarios de espera circular y estrategias para prevenir o detectar interbloqueos en sistemas concurrentes.

3 entregas

Tema 5

Administracion de Memoria

Analisis de paginacion, segmentacion, swap y memoria virtual para entender como el sistema operativo optimiza el uso de RAM entre procesos activos.

3 entregas

Tema 6

Arquitectura del Sistema de Archivos

Estudio de la estructura jerarquica del sistema de archivos, inodos, tipos de archivos y entrada/salida para entender como el kernel administra el almacenamiento y acceso a datos.

3 entregas

Tema 7

Senales

Analisis del manejo de señales en procesos Unix, acciones por defecto y personalizadas, y tratamiento de SIGINT para responder a eventos asincronos sin finalizar el programa.

3 entregas
